= De-Duplicate

Payload parsing (`name:Type` / `when` / `at` / `after`) is implemented
three separate times (`helpers.rs`, `states.rs`, `actions.rs`) with no
shared function -- consistency risk for future changes.

Deduped:
`helpers.rs`'s `parse_accept_trigger` was already the canonical
implementation (used by `parse_accept_action`);
the two inline copies in `states.rs`'s `parse_transition` and
`actions.rs`'s `parse_target_transition` were byte-for-byte duplicates and
are now calls to it.
No behavior change (verified: full test suite passes unchanged).

Bare `ordered` / `nonunique` cardinality-modifier parsing is
copy-pasted across roughly six call sites rather than shared
fragile for future usage kinds that forget to copy the snippet.

Deduped:
found 11 copies total, across both KerML and SysML.
KerML already had two canonical helpers
(`parse_multiplicity_modifiers` in `kerml/definitions.rs`,
`parse_ordering_modifiers` in `kerml/usages.rs`)
that most call sites bypassed with inline loops instead of calling.
Those are now routed through them.
SysML's `usage.rs` had four inline copies and no shared helper at all;
added one (`parse_ordering_modifiers` in `sysml/helpers.rs`) and
pointed all four call sites at it.
The one non-consuming boolean peek
(`kerml/definitions.rs`'s `is_modifier` check in `parse_multiplicity`) was
left alone since it isn't the same pattern.
No behavior change (verified: full test suite passes unchanged).
